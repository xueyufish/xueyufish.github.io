# AGENTS.md

## Project

Jekyll ~> 4.3 static blog (Chinese-language). Hosted via Cloudflare behind `www.yuxiumin.com`. No npm/Node вАФ pure Ruby/Bundler toolchain. Custom domain via `CNAME`.

## Commands

- **Install deps:** `bundle install`
- **Local dev server:** `bundle exec jekyll serve` (incremental builds enabled in `_config.yml`)
- **Production build:** `bundle exec jekyll build`

## Architecture

```
_posts/          Blog posts (YYYY-MM-DD-slug.md)
_drafts/         Unpublished drafts
_layouts/        default.html вЖР page.html вЖР post.html  (post inherits page inherits default)
_includes/       head.html, footer.html (referenced by layouts)
_plugins/        reading_time.rb, search_index.rb (see Plugins below)
assets/css/      12 source files вЖТ main.min.css (bundled output)
assets/js/       8 source files вЖТ main.min.js (bundled output); jquery.min.js loaded separately
search/          Client-side search (index.json generated by plugin)
```

## Post front matter

**Required fields:** `layout: post`, `title`, `date` (YYYY-MM-DD), `author`, `keyword`, `tags`

**Optional fields:**
- `description` вАФ page description for SEO/meta tags
- `og-image` вАФ custom Open Graph image
- `edit-date` вАФ revision date (displays "дњЃиЃҐдЇО" in post header, used in JSON-LD `dateModified`)
- `noindex` вАФ set to `true` to exclude from search engine indexing
- `exclude_from_search` вАФ set to `true` to skip from site search index

## Plugins

- **reading_time.rb** вАФ Counts Chinese characters + English words (strips code blocks first), calculates reading time at 300 words/min. Adds `word_count` and `reading_time` to post data.
- **search_index.rb** вАФ Generates `search/index.json` into `_site/` via `:post_write` hook (runs after build completes). This file is auto-generated вАФ **never edit manually**. If you see stale search results, delete `_site/search/index.json` and rebuild.

## Key conventions

- `version` field in `_config.yml` is used as a cache-bust query param (`?v=вА¶`) on `main.min.css`
- To bust CSS/JS cache after asset changes: update the `version` value in `_config.yml`, then rebuild
- CSS/JS are individually maintained source files in `assets/css/` and `assets/js/`; `main.min.css` and `main.min.js` are the bundled outputs
- CDN dependencies: Font Awesome from `cdn.staticfile.org`, MathJax/anchor-js/FastClick from `cdn.bootcss.com`
- Markdown: kramdown with GFM input; syntax highlighting: rouge
- Dark mode: `data-theme` attribute on `<html>`, toggled via localStorage
- CI (`.github/workflows/cloudflare-cache-purge.yml`) purges Cloudflare cache on push to `main` (also has `workflow_dispatch`)
- `_config.yml` excludes `scripts` dir and `*.gemspec` from build
- `.gitignore` excludes `*.sh` files
- No test suite exists вАФ this is a static site
