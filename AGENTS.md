# AGENTS.md

## Project

Jekyll 4.x static blog (Chinese-language). Hosted via Cloudflare behind `www.yuxiumin.com`. No npm/Node — pure Ruby/Bundler toolchain.

## Commands

- **Install deps:** `bundle install`
- **Local dev server:** `bundle exec jekyll serve` (incremental builds enabled in `_config.yml`)
- **Production build:** `bundle exec jekyll build`
- **CSS/JS asset bundling:** `bash scripts/build.sh` — merges multiple CSS/JS files into `assets/css/main.min.css` and `assets/js/main.min.js`, optionally minifies with `cleancss`/`terser`, and auto-bumps the `version` field in `_config.yml`
- **Cleanup old assets:** `bash scripts/cleanup.sh` — removes outdated CSS/JS files (run with caution)

## Architecture

```
_posts/          Blog posts (YYYY-MM-DD-slug.md, front matter: layout/title/date/author/keyword/tags)
_drafts/         Unpublished drafts
_layouts/        default.html → page.html → post.html
_includes/       head.html, footer.html (referenced by layouts)
_plugins/        Custom Ruby plugins — reading_time.rb (word count / reading time), search_index.rb (generates search/index.json)
assets/css/      main.min.css is the bundled output; individual source files also exist
assets/js/       main.min.js is the bundled output; jquery.min.js loaded separately
search/          Client-side search (index.json generated by plugin, JS consumed)
scripts/         build.sh, cleanup.sh (excluded from Jekyll build via _config.yml)
```

## Key conventions

- Posts use `layout: post` and require front matter fields: `title`, `date`, `author`, `keyword`, `tags`
- The `version` field in `_config.yml` is used as a cache-bust query param (`?v=…`) on CSS assets
- `search/index.json` is auto-generated by `_plugins/search_index.rb` at build time — do not edit manually
- The build script (`scripts/build.sh`) writes to the source tree (not `_site/`), so run it **before** `jekyll build`
- CDN dependencies (Font Awesome, MathJax, anchor-js) are loaded from `cdn.staticfile.org` and `cdn.bootcss.com`
- The CI workflow (`.github/workflows/cloudflare-cache-purge.yml`) purges Cloudflare cache on push to `main`
- `.gitignore` excludes `*.sh` files — the scripts are tracked but any new shell scripts would be ignored by default
